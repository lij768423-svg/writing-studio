## iOS27开放第三方AI - 2026-05-09

### 做得好的
- 搜索阶段用了中英文双搜，Bloomberg/9to5Mac/TechCrunch 一手信源覆盖全，为后续写作提供了扎实的事实基础
- 三遍审校机制确实有用：第一遍就抓住了 iPhone 活跃数错误（20亿→10亿+）、错字（Anrthropic）、时间错误（去年→前几天），这些不改会很致命
- 配图用了 draw.io 直接生成，4 张图从封面到机制图到时间线到竞争格局，和文章结构一一对应，比只写配图需求更完整

### 需要改进的
- 选题讨论阶段，我给的第一个大纲偏窄，用户自己动手重新规划了结构——说明我对选题容量的判断不够，应该更主动地把多个角度整合进主选题里
- 个人素材库（全部即刻动态.csv）不存在，Step 5.5 没执行。这次没有个人经历支撑，文章读起来少了一层「我在现场」的质感
- 初稿写完后发现跟上一篇文章的结尾结构（AI 竞争三阶段）有重复感，虽然改了时间参照但逻辑框架没变——可以考虑每次用不同的收尾方式

### 下次注意
- 选题讨论时，如果用户给了一个「最大容量」的方向，主动把其他角度收进来当分论点，不要让用户自己来重组大纲
- 「前几天的文章」这类跨文章引用要查原文确认时间，不能靠记忆；
- draw.io 长图用 `-s 2` 导出 2x 分辨率，微信公众号显示更清晰
- 前瞻/传闻类选题要在全文每个关键段落都标注「据报道」，不能只在开头和结尾标一次

## 彩礼诈骗入刑 - 2026-05-17

### 做得好的
- 调研阶段挖出了「12 年」的实际来源（马某案而非卢某案），及时纠正了大纲中的错误，避免了事实硬伤
- 三批典型案例时间线梳理清楚，为文章提供了扎实的纵向脉络
- 用 mmx CLI 生成配图，四张 AI 图片风格统一，比纯示意图更有视觉冲击力

### 需要改进的
- 法律题材对措辞精确度要求远超科技题材。五处修改全集中在同一类问题：把司法信号写成了法律变化、把个案程序拔成了普遍义务、从公开材料做了过度推断
- 结尾「彩礼是美好传统」容易引发读者分流争议，法律题材应避免对传统做价值判断
- mmx CLI 输出的图片实际是 JPEG 格式但自动存为 .png，造成扩展名与实际格式不一致，需要手动修正
- mmx CLI 已更名为 `mmx`（MiniMax 官方 CLI，路径 `/opt/homebrew/bin/mmx`），旧笔记里的「mmx CLI」现在用 `mmx <resource> <command>` 调用

### 下次注意
- 法律题材的核心原则：把「今年最大的变化」改成「信号更明确了」；把「有义务」改成「可以」；把系统机制推断改成「可能触发注意」
- 不要在公开案例材料基础上脑补操作细节（如「系统亮红灯」），法律读者会逐句挑刺
- 结尾不做价值升华，不给传统贴标签，回到核心判断即可
- 用 mmx 生成图片后，先用 `file` 命令检查实际格式再插入文章

- 01-research.md 没显式列来源 URL，orchestrator 报"未检测到来源链接"——下次写 research 立刻加 http 链接

## 工作流维护 - 2026-06-03

### 做得好的
- 工作流分档化（fast / standard / strict）已稳定运行一个多月，standard 1 轮 review 适合绝大多数公众号文章
- 选题讨论 → 写大纲 → self-review → 修订 → 初稿 → 三遍审校 → 归档的链路顺

### 需要改进的
- **codex MCP 不可靠**：`codex` 是 node 脚本，启动时内部 spawn 找不到 `node`（shell PATH 不带 `/opt/homebrew/bin`），外层 PATH 注入能解决但太脆
- 维护 codex-review 模板 + runCodex 函数 ≈ 110 行代码，性价比低
- 弃用 codex 改为 self-review（Claude 按 codex 视角写敌对审视），少 1 个外部依赖、少 1 类故障

### 下次注意
- 凡是依赖外部 CLI agent 的设计，要先在 shell PATH 缺失场景下试一遍，别假定 `command -v` 能找到
- 选 review 流程时优先选「主 agent 自我审视」方案，少外部依赖、少协调成本——除非真的有强需求要"敌对 agent 视角"
- 弃用某个工具前先 grep 全工作区（`grep -rln "<tool>" --include="*.md" --include="*.cjs" --include="*.json"`）确认删除清单，避免漏改

## 全球AI竞赛地缘政治经济学 - 2026-06-03

### 做得好的
- 调研阶段用了 tvly（5/30 后默认搜索源）+ 中文/英文双搜 + 明确标注待核实项；VOA / 欧盟理事会 / 经济观察网 / 新华网瞭望四类信源覆盖美 / 欧 / 中三个角度，为政治层的"三套规则"提供事实底座
- 大纲用了"理论框架 + 三层结构 + 三个案例 + 结尾"的清晰骨架，每张幻灯片标了"~X 分钟"和主题，演讲时不用动脑算时间（沿用 5/30 的节奏标注法）
- 三遍审校发现 6 处具体修订点（"一年前→半年前"、"三重→四重定价"、"寒武纪"列举补"百度昆仑芯"、"最后一张大牌"软化、"民主国家"加引号），这些硬伤如果不查会跟着 PPT 流出去
- 复用了 5/30 / 6/3 两篇的素材但**显式不重复案例**——开头结尾都点明本稿的"全球结构"定位，避免自我抄袭

### 需要改进的
- **orchestrator 的 claude-research / claude-draft 命令有 PATH 问题**：node spawnSync('claude', ...) 在 Bash 工具继承的 PATH 里找不到 /opt/homebrew/bin/claude，跟 6/3 弃用 codex 是同一类问题。3 次失败都返回 `status: null`、空 stdout/stderr，调试花了一轮对话
- 调研阶段用了"claude-research 失败 → tvly 搜索"两步，绕了一圈；下次直接用 tvly（`export PATH=...$HOME/.local/bin:/opt/homebrew/bin:$PATH`），不要再试 orchestrator 的 claude-* 包装
- 6/3 教训说"凡是依赖外部 CLI agent 的设计，要先在 shell PATH 缺失场景下试一遍"——但 orchestrator 的 claude-research 没遵守这条；写 orchestrator 时应当用绝对路径 + 兜底 PATH 注入，而不是 spawn 一个未知 PATH 环境的子进程
- tvly / claude / node 都不在默认 PATH 上，这台机器的所有 spawnSync 都要用 `/opt/homebrew/bin/...` 或 `~/.local/bin/...` 绝对路径——这个发现应该写进 `WORKFLOW.md` 的"环境要求"一节，避免下次再踩

### 下次注意
- orchestrator 的 `claude-research` / `claude-draft` / `claude-revise` 命令当前是脆的（PATH 问题），但 `init` / `validate` / `archive` / `mark-reviewed` 状态机命令是稳的。**直接执行"init → 自己写 research/outline/draft → mark-reviewed / finalize / validate / archive"** 是 standard 档的最佳路径
- 跨篇引用（"5/30 稿讲过""6/3 稿不重复"）要在 PPT 开头和结尾**都点明**——读者不会自动关联到 4 天前 / 3 天前的文章
- 政治经济学类选题，**理论框架要在最前面 3 张幻灯片铺好**（马克思 + 沃勒斯坦），后面讲经济 / 科技 / 政治时反复回引，读者才跟得上"剥削"这个核心词
- 经济类数据（4 月中国集成电路出口 +100.1%、Anthropic 付 xAI $15B/年）要标年份月份，避免"最近"这种模糊词——读者会问"你写的时候数据是什么时点的"
- "反中心"这种金句要搭配具体路径（开源 / 国产替代），否则就是空话；这次"开源是 AI 时代的反中心武器"配了 DeepSeek / Qwen / GLM 的实例，可读
