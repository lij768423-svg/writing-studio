# 素材索引

> 将你的即刻动态、微博、推特等内容按主题整理到这里，方便 AI 快速查找真实素材。
> 当前索引先收录本工作区已有的写作参考、爆文拆解和流程文档。后续导入即刻、微博、公众号旧文后，再继续按主题扩展。

## 使用原则

写作前优先按这个顺序找素材：

1. 先看风格：确认文章应该怎么开头、怎么组织、哪些表达不能用。
2. 再看高流量参考：学习标题、结构、开头、转发理由。
3. 再找个人素材：优先使用真实经历、真实观点、真实测试结果。
4. 最后查知识库：涉及新产品、新技术、新数据时，必须用最新资料验证。

素材使用底线：

- 不能把别人的经历写成“我”的经历。
- 不能把网上资料改写成亲身实测。
- 不能为了文章好看编造数据、截图、用户反馈。
- 可以学习别人的结构和问题意识，但不能复制标题、段落和表达。

## 选题时怎么用这个索引

如果要写 AI 工具实测：

1. 读 [风格指南](风格指南.md)，确认语言节奏。
2. 读 [高流量文章参考](高流量文章参考.md) 里的工具横评样本。
3. 查 `_knowledge_base/` 是否已有该工具资料。
4. 如果没有，先搜索并建立知识文件。
5. 用真实测试数据写，不用官方介绍凑正文。

如果要写公众号方法论：

1. 读 [高流量文章参考](高流量文章参考.md) 里的爆文方法和内容号复盘。
2. 回到 [风格指南](风格指南.md)，避免写成课程腔。
3. 优先加入自己的写作流程、审校经验、真实案例。
4. 结尾给可执行清单，不做空泛升华。

如果要写商单：

1. 先保存 brief 到 `_briefs/`。
2. 查品牌、产品、竞品资料，保存到 `_knowledge_base/`。
3. 用 [风格指南](风格指南.md) 控制口吻。
4. 用 [高流量文章参考](高流量文章参考.md) 学结构，但不要套标题。
5. 审校时检查有没有硬夸、编造和过时信息。
