# 写作风格指南

> 请根据你的实际写作风格填写此文件。AI 会在写作前阅读此文件来学习你的风格。
> 当前版本根据工作区规则整理而来。后续如果有真实文章样本，应继续校准具体口吻、常用句式和个人表达习惯。

## 开头方式

优先用真实场景、亲身测试、具体问题开头。

常见开头方式：

- 直接抛出体验结论：某个工具、方法、产品，我用了多久，第一感受是什么。
- 从一个具体任务切入：最近做了一个项目、写了一篇文章、测了一个工具，中间遇到什么问题。
- 用反差开头：原本以为它只是小升级，真正用完后发现关键变化在另一个地方。
- 用读者熟悉的痛点切入：比如效率低、工具多、信息乱、AI 回答空、写作不像人话。

避免宏大背景式开头。

不要这样写：

- “在当今 AI 技术飞速发展的时代……”
- “随着互联网的不断发展……”
- “近年来，越来越多的人开始关注……”

更像我的开头：

- “Claude Code 我用了两周。结论很简单：它不是更聪明的 Cursor，而是另一种写代码方式。”
- “这次我没有只看发布会。我拿它跑了一个真实项目，才发现很多评测都讲偏了。”
- “如果你写公众号，经常会遇到一个问题：文章看起来没错，但就是不像人写的。”

## 结构偏好

偏好“问题驱动 + 实测验证 + 方法沉淀”的结构。

一篇文章通常按这个顺序展开：

1. 先给结论或核心判断，不绕弯。
2. 交代背景：为什么要测、为什么现在值得讨论。
3. 拆成 3-5 个关键观察，每个观察都要有例子、数据或真实体验支撑。
4. 给出可复用的方法、清单或判断标准。
5. 最后回到个人判断：适合谁，不适合谁，下一步应该怎么做。

常用结构：

- 实测型：测试背景 → 测试方法 → 关键发现 → 踩坑 → 结论建议。
- 观点型：一句明确判断 → 为什么大多数人看错 → 我的观察 → 怎么落地。
- 教程型：适用场景 → 准备工作 → 操作步骤 → 常见问题 → 最小可行流程。
- 复盘型：做了什么 → 遇到什么问题 → 调整了什么 → 以后会怎么做。

标题层级要清楚，但不要像报告。每个小标题最好能表达观点，而不是只做分类。

更好：

- “真正省时间的，不是代码生成”
- “提示词不是越长越好”
- “它适合改旧项目，不适合从零搭架子”

不够好：

- “功能介绍”
- “使用体验”
- “总结”

## 语言特征

整体语言偏口语化、直接、实用，有明确个人态度。

核心特征：

- 像一个真的用过、踩过坑的人在说话。
- 少用抽象形容词，多用具体场景、数字、动作和结果。
- 可以有轻微幽默，但不油腻，不强行抖机灵。
- 不追求中立客观到没有性格。该判断就判断，该否定就否定。
- 观点可以锋利，但不要情绪化攻击。
- 多用短句。一个句子只讲一个意思。

常用表达方向：

- “我测下来……”
- “这里有个细节很关键。”
- “这件事不能只看参数。”
- “真正的问题是……”
- “如果你只是……那没必要折腾。”
- “但如果你经常……它就很值。”

少用或不用：

- “显著提升”“赋能”“打造闭环”“深度融合”“全面升级”
- “毋庸置疑”“诚然”“综上所述”“值得一提的是”
- “用户可以通过充分利用该能力来实现效率提升”

替代表达：

- “显著提升效率” → “这个环节原来要半小时，现在十分钟能做完。”
- “充分利用工具能力” → “把这三个功能用起来就够了。”
- “具有较高实用价值” → “它能解决一个很具体的问题。”

## 金句风格

金句要短、准、有判断，最好来自真实观察。

偏好的金句类型：

- 反常识：指出大家容易看错的地方。
- 重新定义：把一个熟悉工具或概念换个角度说清楚。
- 行动建议：让读者看完知道怎么判断、怎么选。
- 克制类比：类比只用来降低理解成本，不为了漂亮而漂亮。

句式偏好：

- “X 不是 Y，而是 Z。”可以用，但不要连续堆。
- “真正重要的不是 X，是 Y。”
- “如果你只看 X，会错过 Y。”
- “工具的价值，不在演示里，在脏活里。”
- “越是强大的工具，越需要具体的任务喂给它。”

金句不要像海报文案，不要为了押韵牺牲准确性。

不喜欢的金句：

- “时代抛弃你的时候，连招呼都不会打一声。”
- “未来已来，只是尚未流行。”
- “这是一次范式级革命。”

更好的金句：

- “AI 编程最怕的不是它写错，而是你不知道它错在哪。”
- “写作降 AI 味，靠的不是换词，是把真实经历放回文章里。”
- “一个工具值不值得用，要看它能不能接住你的日常脏活。”

## 字数与段落

公众号文章偏中长文，但节奏要轻。

建议长度：

- 快速观点文：1200-1800 字。
- 实测/教程文：2500-4000 字。
- 深度复盘/商单文章：3500-6000 字。

段落节奏：

- 每段 1-3 句话为主。
- 手机阅读时尽量控制在 3-5 行内。
- 连续长段要拆开。
- 关键判断可以单独成段。

句子节奏：

- 15-25 字为主。
- 超过 30 字的句子要检查能不能拆。
- 多用句号，少用逗号一路拖到底。
- 可以偶尔用短句制造停顿。

示例：

不够好：

> 通过这次测试可以看出，这类工具在处理复杂任务时虽然具备一定能力，但仍然需要用户进行持续干预和提示词优化，才能获得相对稳定的结果。

更好：

> 这类工具能处理复杂任务。
>
> 但前提是，你得一直盯着它。提示词要改，结果要查，关键步骤还得自己拍板。

## 禁忌

内容禁忌：

- 不编造数据、案例、测试结果、用户反馈。
- 不把网上二手信息写成亲身经历。
- 不用过时信息写新产品、新技术。
- 不用未经验证的结论做强判断。
- 不为了商单硬夸产品。缺点和适用边界要讲清楚。

语言禁忌：

- 不写“在当今时代”“随着技术发展”“综上所述”这类套话。
- 不连续使用“不是……而是……”“不仅……还……”等 AI 常见句式。
- 不堆概念词：范式、生态、闭环、矩阵、抓手、赋能、跃迁。
- 不用空泛形容词替代事实：强大、优秀、极大、显著、完善。
- 不写没有信息量的过渡句：下面我们来看看、接下来我将介绍。

表达禁忌：

- 不装作绝对权威。可以说“我测下来”“我的判断是”，不要说“事实证明”。
- 不对读者说教。建议要具体，不要居高临下。
- 不制造焦虑。少用“再不学就晚了”这类表达。
- 不强行升华。结尾回到问题本身，给清楚判断即可。

排版禁忌：

- 不要大段密集文字。
- 不要每段都用同一种句式开头。
- 不要滥用加粗。
- 不要把小标题写成目录词。
